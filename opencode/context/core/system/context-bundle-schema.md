# Context Bundle Schema

Purpose: canonical schema for `.tmp/context/{session-id}/bundle.md` used in delegations.

Use the exact section order:

## Project Summary
- High-level project context relevant to this task
- Architecture notes needed for decisions

## Relevant Files
- Paths and short reason why each file matters
- Include only files required for this task

## Current Task
- Objective, expected output, and done criteria
- Explicit scope boundaries

## Constraints
- Standards, policies, and non-functional requirements
- Validation and approval constraints

## Open Questions
- Unknowns that can block implementation/review
- Mark `none` if there are no open questions
