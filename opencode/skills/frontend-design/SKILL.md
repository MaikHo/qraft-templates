This skill is for .NET UI work only. Use it when creating or refining interfaces
in Blazor or AvaloniaUI.

## Workflow

1. Define UX goal and target platform (Blazor/Avalonia)
2. Choose a clear visual direction
3. Design component/view structure and state flow
4. Implement incrementally in C#/.NET UI files
5. Validate with `dotnet build` and relevant tests

## Rules

- Keep business logic out of UI layers
- Favor reusable components and clear parameters
- Follow accessibility basics (focus, contrast, keyboard navigation)
- Use design tokens/resources for consistent typography, spacing, and colors
- Avoid ad-hoc one-off styling and magic values

## Output Expectations

- Files changed
- Design rationale
- Validation results
- Next iteration suggestions (if needed)
